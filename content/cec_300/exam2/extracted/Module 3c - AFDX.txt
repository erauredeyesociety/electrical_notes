VL Scheduling
•
Located at the End Systems
o
Traffic shaping via VL Scheduler
o
Multiplexes frames from all VLs onto the physical layer
o
VL provides a stream of frames.
•
Regulated by the Bandwidth Allocation Gap (BAG)
•
Minimum space between messages
o
1 – 128 ms (value must be power of 2)
o
Jitter is the upperbound on transmit latency (frame time offset)

Communication Port Types
•
Receive Sampling Ports
•
Buffer sufficient for one message.
•
Incoming messages overwrite previous 
message
•
Read at any time, always receive the 
most recent message.
•
Transmit Sampling Ports
•
Continuously transmits the most recent 
message until new message received.
•
Then, new message is continuously 
transmitted.
•
Receive Queueing Ports
•
FIFO queue for received messages
•
Application can read at any time, 
dequeuing and returning the message 
at the front of the queue.
•
Transmit Queueing Ports
•
Transmits new messages in the queue 
port as fast as it can.
•
Each message is dequeued before 
transmission.
•
New messages are appended to the 
back of the queue.
Image Source: https://web.archive.org/web/20150618140031/http://www.techsat.com/fileadmin/media/pdf/infokiosk/TechSAT_TUT-AFDX-EN.pdf

Integrity Checking / Redundancy Management
•
Transmitting Mode:
o
Each frame is given a number from 0-
255 with each frame number being 
1+previous frame’s SN
•
Receiving Mode:
o
The IC algorithm accepts all frames that 
comply with one of the following 
criteria:
▪
SN = 0 (The transmitting ES is 
started or reset)
▪
SN = Previous SN + 1
▪
SN = Previous SN + 2
▪
All frames not complying with 
these criteria are discarded.
•
SN = Sequence number (8-bit)
•
Unique to each VL
•
Integrity Checking
o
Done for each network, one per VL, of End System 
nodes
o
Independent of Redundancy Management
o
Applied on the MAC layer using the SN of the 
ethernet frame
Image Source: https://web.archive.org/web/20150618140031/http://www.techsat.com/fileadmin/media/pdf/infokiosk/TechSAT_TUT-AFDX-EN.pdf

Switch Architecture
•
Filtering
o Confirm destination address
o Received message on valid 
destination port
o FCS validity
o Ethernet frame size alignment
o Ethernet frame size range
o Adherence to MTU (maximum 
transfer unit in bytes) per VL
•
Traffic Policing
o Byte-based: filters out VL in 
terms of bandwidth usage
o Frame-based: filters out VL in 
terms of bandwidth in frames 
per second.
Image Source: https://web.archive.org/web/20150618140031/http://www.techsat.com/fileadmin/media/pdf/infokiosk/TechSAT_TUT-AFDX-EN.pdf 
•
Virtual Link Routing – uses a 16-bit Virtual Link ID (VL ID) with frame 
as address for End System nodes (unlike MAC, used for Ethernet)
•
Configuration Tables – define routing paths including input ports, 
output ports of a given VL ID.
