• Core Components
o Proportional (P) (𝑲𝒑): Reacts to the current error. If the error is large, the control action is large. 
• It provides immediate reaction but cannot eliminate the error on its own.
o Integral (I) (𝑲𝒊): Accounts for past errors. 
• It accumulates the error over time, forcing the system to eliminate the residual steady-state error that a P-
controller cannot.
o Derivative (D) (𝑲𝒅): Predicts future errors based on the rate of change.
•  It acts as a damper, reducing overshoot and improving stability

What does the proportional element of 
the controller do?
The proportional unit is the largest driver of 
change within the control system.  
Simply put, its output is the error multiplied 
by the proportional coefficient.
Limitation: Steady-state error. 
As the error shrinks, the amount of 
power to deliver the change persists 
resulting in a solution that is near, but 
not quite at the commanded state.

What does the integral do?
The integral is the accumulated error at the 
current time step.  i.e., it sums the error 
over time.
If error persists (even small error) from the 
proportional controller, the integral 
becomes larger meaning there is a greater 
“push” driving the error toward zero.
However, if the value of 𝐾𝑖 is too high, 
either the system becomes sluggish or it 
begins to overshoot.

What does the derivative do?
The derivative acts like a dampener or 
limiter, which limits the rate of change 
more as the output reaches the set point, 
i.e. as error approaches zero.
This mitigates cases where the controller’s 
response overshoots the set point.
However, if the measurements/state 
estimates are noisy, which appear as small 
but significant change in error, which can 
cause the control system to likewise 
twitch/jitter.

What does Feed Forward do?
It serves several purposes:
•
Reduces lag in response because the 
control’s output will be tuned around 
the commanded state.
e.g. If the vehicle needs to slow by 0.5 
of throttle’s airspeed command, then the 
feed forward path will add the target, 0.5 
as a starting point for the output and the 
PID will fine tune the system response.
Note that the feedforward terms are not 
part of the PID control loop.

What does  𝑻𝒕𝒓𝒊𝒎 do?
It represents the desired steady state for 
the controller’s output. 
As startup, with 𝑇𝑡𝑟𝑖𝑚 and the 𝐾𝐹𝐹, dials in 
the expected state to achiever the desired 
set point.
This reduces the efforts of the PID 
controller to identifying the small 
corrections that need to be made.
It effectively puts the center of the 
controller’s range at a particular value such 
that the controller does not become 
saturated.
A saturated controller is similar to a saturated reaction wheel.
It has made enough correction to the maximum or minimum 
such that it can no longer increase or decrease.

Tuning PID-based Control System
•
Tuning for a small UAS is typically achieved through an experienced pilot flying the 
vehicle and making observations.
•
First tune for stable level flight
•
Then tune for lateral and longitudinal flight by adjusting the gains of the PID 
controller.
•
Some level of tuning can be handled through simulation
•
Software in the Loop
•
Simulate environment, aircraft dynamics model, and runs virtual control 
system
•
Sensor data is simulated based upon simulation state
•
Command influences vehicle in simulation given dynamics model 
•
Hardware in the loop
•
Similar to SIL
•
Hardware controller is connected, but utilizing simulation’s sensor inputs and 
commands are handled by simulation and not actual aircraft systems

Contingency Management
•
Autonomous operations to support the 
resolution of a safety critical failure
o Attempts to mitigate risk, and
o Allow time for recovery
•
Examples:
o Lost communications
•
Fly to rally point and loiter
•
Fly home and land
o Onboard fire
•
Fly to suitable landing site and land
o Lost GPS
•
Loiter until manual control
•
Capabilities can be built in
o Some systems have simple automated routines
o Others have complex “trees” of actions based 
upon mission and pre-planned contingency actions
