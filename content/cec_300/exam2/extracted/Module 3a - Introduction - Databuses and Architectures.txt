--- Page 16 ---
•
Application Layer
o Requests issued to presentation layer
o Receives responses to requests from the presentation 
layer
o Examples: HTTP, FTP, POP, etc.
•
Presentation Layer
o Provides a common interface to a variety of different 
application layer entities.
o Enables encryption/decryption and 
compression/decompression
•
Session Layer
o Establishes, manages, and terminates sessions
o Enables synchronization (e.g. streaming)
•
Transport Layer
o Responsible for the transportation of data packets 
between entities.
o Port addressing, segmentation/reassembly, connection 
management, reliability and error control.
o e.g. Transportation control protocol (TCP) and User 
Datagram Protocol (UDP)
•
Network Layer
o Constructing data packets
o Routing
o Local addressing (e.g. Internet Protocol (IP))
o Traffic control and congestion management
•
Data Link Layer
o Node-to-Node data transfer
o Organizing raw physical bits into frames
o Medium Access Control (MAC) addresses for local 
delivery
o Error control (e.g. CRC)
•
Physical Layer
o Bits to signals / signals to bits
o Physical medium & topology
o Data rates
o Transmission modes
o Hardware components (Cables, network cards, hubs, 
repeaters, etc.)
